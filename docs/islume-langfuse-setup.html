Self-Hosted Setup Guide

Islume + Langfuse
Local Development로컬 개발 환경

A guide to running the Islume microservices alongside a self-hosted Langfuse v3 observability stack on a local machine. Docker handles the infrastructure (Redis, PostgreSQL, ClickHouse), so the actual install footprint is small. Islume 마이크로서비스와 셀프호스트 Langfuse v3 옵저버빌리티 스택을 로컬 머신에서 함께 실행하는 가이드. Docker가 인프라(Redis, PostgreSQL, ClickHouse)를 담당하므로 실제 설치 항목은 많지 않다.

Langfuse v3
Self-Hosted LangfuseLangfuse 셀프호스트 설치
Langfuse ships its own docker-compose.yml bundling all infrastructure (PostgreSQL, ClickHouse, MinIO, Redis). Run it from a separate directory and remap host ports to avoid clashing with Islume — see the port policy below. Langfuse는 자체 docker-compose.yml에 모든 인프라(PostgreSQL, ClickHouse, MinIO, Redis)를 포함한다. 별도 디렉토리에서 실행하고, Islume과의 충돌을 피하기 위해 호스트 포트를 재매핑한다 — 아래 포트 정책 참고.

Add the port-remapping override포트 재매핑 override 추가

Drop the docker-compose.override.yml (shown in the port policy below) into the repo root. Compose merges it with the base file automatically, so the original is never touched (no conflicts on git pull).아래 포트 정책에 있는 docker-compose.override.yml을 레포 루트에 넣는다. Compose가 base 파일과 자동 병합하므로 원본은 건드리지 않는다(git pull 시 충돌 없음).

Langfuse — Port Policy
Port Remapping for Coexistence공존을 위한 포트 재매핑
Islume and Langfuse both ship Docker Compose stacks that default to the same host ports (PostgreSQL 5432, Redis 6379, etc.). To run both on one machine, Langfuse's host-side ports are shifted. Container-internal communication keeps the original ports, so Langfuse itself is unaffected — only host access uses the new ports. Islume과 Langfuse 모두 동일한 호스트 포트(PostgreSQL 5432, Redis 6379 등)를 기본값으로 쓰는 Docker Compose 스택을 제공한다. 한 머신에서 둘 다 실행하기 위해 Langfuse 쪽 호스트 포트를 시프트한다. 컨테이너 내부 통신은 원래 포트를 그대로 사용하므로 Langfuse 동작에는 영향이 없고, 호스트에서 접근할 때만 새 포트를 쓴다.

!override

Use the !override tag (Compose v2.24+) so the base port list is replaced, not appended. Without it, the base mappings (5432/6379/3000/9090) stay live and clash with Islume again.!override 태그(Compose v2.24+)를 사용해 base의 ports 리스트를 append가 아닌 replace로 바꿔야 한다. 안 그러면 base 매핑(5432/6379/3000/9090)이 살아있어 다시 Islume과 충돌한다.

docker-compose.override.yml — place in the Langfuse repo root: docker-compose.override.yml — Langfuse 레포 루트에 배치:

docker-compose.override.yml
services:
  langfuse-web:
    ports: !override
      - "3100:3000"

  postgres:
    ports: !override
      - "127.0.0.1:5433:5432"

  redis:
    ports: !override
      - "127.0.0.1:6380:6379"

  clickhouse:
    ports: !override
      - "127.0.0.1:8124:8123"
      - "127.0.0.1:9001:9000"

  minio:
    ports: !override
      - "127.0.0.1:9092:9000"  # S3 API
      - "127.0.0.1:9093:9001"  # Console UI
Bind to localhostlocalhost 바인딩

Infra ports are bound to 127.0.0.1 so only the local machine can reach them. Only langfuse-web (3100) is left open for browser access.인프라 포트는 127.0.0.1에 바인딩되어 로컬 머신에서만 접근 가능하다. 브라우저 접근용 langfuse-web(3100)만 열어둔다.
